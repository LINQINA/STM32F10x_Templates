# 🧠 RAM 与 Flash 机制全解析（适用于 STM32 初中高级开发者）

---

## ✅ 一、RAM 与 Flash 的本质区别

| 项目    | RAM（数据存储器） | Flash（程序存储器） |
|---------|------------------|---------------------|
| 类型    | 易失性内存         | 非易失性存储器         |
| 掉电是否丢失 | ✅ 是              | ❌ 否                  |
| 是否可写 | ✅ 任意读写        | ❌ 写入需擦除（页/扇区）|
| 读写速度 | ⚡ 极快（单周期）    | 🐢 相对较慢（多周期）   |
| 常见用途 | 变量、栈、堆、缓冲区 | 代码、常量、初始化数据  |

---

## ✅ 二、RAM 分区详解

STM32 RAM 是从 `0x20000000` 开始，具体分区如下：

```
低地址
0x20000000 ─┬─ .data   → 已初始化的全局/静态变量（从 Flash 拷贝过来）
            ├─ .bss    → 未初始化的全局/静态变量（上电自动清零）
            ├─ heap    → 动态分配内存（malloc）
            ├─ ...     → 空闲区域
            └─ stack   → 函数调用栈（从高地址向下增长）
高地址
```

### 📌 FreeRTOS 内存使用：
- FreeRTOS 常通过 `.bss` 中定义一块大数组作为堆（heap）
- 任务栈大小 = `任务字段大小 × 4`（因 Cortex-M 是 32bit 架构，每栈单元4字节）

## ✅ 三、Flash 分区与内容

STM32 Flash 一般从 `0x08000000` 开始，内容如下：

| 地址范围        | 内容            | 说明 |
|------------------|-----------------|------|
| `0x08000000`     | 启动向量表       | 栈顶地址、Reset_Handler 等中断入口地址 |
| `0x08000008` 后  | `.text` 段      | 代码段，函数、语句等机器码指令 |
| `.rodata`        | 只读常量数据段   | `const` 字符串、查找表等 |
| `.data` init     | `.data` 初始值  | 运行前从这里复制到 RAM 的 .data 区 |

## ✅ 四、MCU 启动流程（上电或复位后）

上电后，ARM Cortex-M 内核执行固定流程：

| 步骤 | 操作 |
|------|------|
| Step 1 | 从地址 `0x08000000` 读取 MSP（栈顶指针） → 加载到 SP |
| Step 2 | 从地址 `0x08000004` 读取 Reset_Handler 地址 → 加载到 PC |
| Step 3 | 跳转执行 Reset_Handler（启动代码） |
| Step 4 | 初始化 `.data`、`.bss`，然后进入 `main()` 函数 |

✅ Flash 的前两个地址就是 MCU 的启动引导点（向量表头部）

```

```

## ✅ 六、程序运行时：代码与数据如何协同

| 执行对象   | 存储位置 | 访问方式 |
|------------|----------|----------|
| 指令 fetch | Flash    | PC 每次从 Flash 中取出 32bit 指令 |
| 全局变量   | RAM (.data/.bss) | 通过固定地址读写 |
| 栈变量     | RAM stack | 相对栈指针偏移访问 |
| const 数据 | Flash    | 使用 LDR 常量池或立即数访问 |
| malloc 数据| RAM heap | 动态分配，来自 `.bss` 定义的堆区 |

MCU运行的时候，是把hex文件(二进制)执行的，代码运行的时候,会调用RAM的数据

## ✅ 七、BootLoader 跳转 App 时的三大必要步骤

```c
__set_MSP(app_sp);              // Step 1：设置主栈指针
SCB->VTOR = APP_ADDRESS;        // Step 2：重定向中断向量表（推荐）
((void(*)(void))app_entry)();   // Step 3：跳转至 Reset_Handler
```

## ✅ 八、金句速记（推荐背诵）

> ✅ RAM：运行时读写变量使用，掉电丢失  
> ✅ Flash：存代码与常量，掉电保留  
> ✅ MCU 启动：读 Flash[0] 得 MSP，读 Flash[1] 得 Reset_Handler → 开始执行程序  
> ✅ 跳转执行：设置 MSP → 跳转 PC → 重定向中断向量表
