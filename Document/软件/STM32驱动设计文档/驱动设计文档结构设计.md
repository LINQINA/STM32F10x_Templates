# 驱动设计文档结构设计

本文档定义了 STM32 驱动模块设计文档的统一结构模板。每个驱动模块（DevicesXxx）均按此结构编写文档，保证风格一致、便于查阅。

---

## 0. 概览

每份驱动文档的开头用属性表快速说明模块信息：

| 属性 | 内容 |
| :--- | :--- |
| **模块名称** | DevicesXxx (模块用途简述) |
| **源文件** | `DevicesXxx.c` / `DevicesXxx.h` |
| **硬件依赖** | 芯片型号、外设、引脚 |
| **软件依赖** | 依赖的其他模块（如 DevicesQueue、DevicesDelay） |
| **版本** | vX.X |
| **最后更新** | 日期 |

概览部分需要覆盖：

- **驱动目标**：解决什么问题（如 LED 控制、串口通信、SPI Flash 读写）
- **对外接口**：提供哪些 API（Init / Deinit / Read / Write / Ioctl）
- **依赖关系**：用到哪些外设、GPIO、中断、DMA、FreeRTOS 组件
- **目录结构**：`DevicesXxx.c/h` 的存放位置（BSP 目录）

---

## 一、硬件驱动理解

写成可查表的形式，方便回查。

### 1.1 硬件连接与信号

- 引脚表（端口 / 引脚 / 复用功能 / 上下拉 / 速度）
- 外设框图（MCU 与外设的连接关系）
- 关键电路说明：上拉/下拉、电平转换、供电、晶振、终端电阻等

### 1.2 工作原理

- 外设的核心机制：如按键消抖、SPI 模式、I2C 时序、USART 采样

### 1.3 时序 / 寄存器要点

- 只写"必须理解"的部分：如 SPI CPOL/CPHA、UART 波特率计算、TIM 计数关系
- 不需要搬运参考手册原文，提炼关键点即可

---

## 二、驱动代码编写

分三层写清楚：**接口设计 → 内部实现 → 关键点解释**

### 2.1 对外 API 设计

- `DevicesXxx_Init()`
- `DevicesXxx_Read()` / `DevicesXxx_Write()`
- `DevicesXxx_Ioctl()`（可选）
- 返回值 / 错误码规范（如 0 成功，<0 错误）

### 2.2 状态机 / 资源模型

- 是否需要互斥（串口 / RS485）
- 是否使用中断 / DMA
- 缓冲区设计：环形队列（DevicesQueue）或其他方案
- 超时机制：轮询延时、RTOS 信号量

### 2.3 配置项与宏

- 引脚宏、通道宏、是否开启 DMA / 中断的宏
- 建议统一放到 `DevicesXxx.h` 的配置区

### 2.4 核心代码片段

- 不用整份贴，贴核心路径代码：Init、中断回调、收发函数

## 三、实际调试记录

采用固定的"可复盘模板"格式：

| 字段 | 内容 |
| :--- | :--- |
| **现象** | 观察到了什么（如收不到数据、偶发丢包） |
| **复现条件** | 如何稳定复现（频率 / 数据长度 / 触发步骤） |
| **定位过程** | 做了哪些验证（示波器 / 逻辑分析 / 打印 / 单步） |
| **根因** | 最终原因（配置 / 时序 / 中断优先级 / 缓存覆盖） |
| **修复** | 改了哪里，为什么 |
| **回归测试** | 如何证明修复有效（压测次数 / 条件） |

长期积累后，这部分会形成个人的 Debug 手册。

## 文档目录约定

每写一个驱动，复制本模板填入内容即可。文档统一放在：


目前已有文档：

- `DevicesUart_Design.md`
- `DevicesKey_Design.md`
- `DevicesRS485_Design.md`
- `DevicesSPI_Design.md`
- `DevicesSPIFlash_Design.md`
- ...

所有文档按相同结构编写，方便横向对比和快速定位。
